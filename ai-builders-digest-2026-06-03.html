Forge: Guardrails Take an 8B Model from 53% → 99.3% on Agentic Tasks
TI Research
May 19, 2026 · ⭐ 687 HN points · 252 comments

Antoine Zambelli, AI Director at Texas Instruments, built Forge — an open-source reliability layer for self-hosted local LLMs. The results are staggering: an 8B model goes from 53% to 99.3% on multi-step agentic workflows just by adding the right architectural guardrails — no model changes needed.

  • Ministral 8B + Forge reaches 99.3% — outperforming Claude Sonnet without guardrails (87.2%)
  • Five guardrail layers: retry nudges (24–49 point drop when disabled), error recovery (~10 points), step enforcement, rescue parsing, context compaction
  • Discovered that serving backend matters more than model: same 12B weights produced 7% on llama-server vs 83% on llamafile — a 75-point swing
  • Introduced ToolResolutionError — distinguishing "tool returned data" from "tool found nothing" for the first time in LLM tool-calling
  • Paper accepted to ACM CAIS '26, 97 model/backend configs, 18 scenarios, 50 runs each

"The gap between a free local 8B model on a $600 GPU and a frontier API is less than 1 point. An 8B model with framework support beats the best result you can get through frontier API alone."

— Antoine Zambelli, AI Director, Texas Instruments

Needle: 26M Parameter Tool-Calling Model Distilled from Gemini
Cactus Compute
May 12, 2026 · ⭐ 776 HN points · 211 comments

Cactus Compute open-sourced Needle, a groundbreaking 26 million parameter model for function calling that runs at 6,000 tok/s on consumer devices. The key insight: tool calling is retrieval-and-assembly, not reasoning — so they removed all FFN layers and built a "Simple Attention Network" that outperforms models 10-20x its size.

  • Entirely cross-attention + gating — zero MLP/FFN layers in the architecture
  • Pretrained on 200B tokens (16 TPU v6e, 27 hrs), post-trained on 2B tokens of synthesized function-calling data (45 min)
  • 6000 tok/s prefill and 1200 tok/s decode on consumer devices — runnable on phones
  • Beats FunctionGemma-270M, Qwen-0.6B, Granite-350M on single-shot function calling
  • "No FFN" finding generalizes to any task with external structured knowledge (RAG, tool use)

"Massive models are overkill for tool calling. Tool calling is fundamentally retrieval-and-assembly — match query to tool name, extract argument values, emit JSON. Cross-attention is the right primitive for this."

— Henry Ndubuaku, Cactus Compute

Batch API Is Terrible for One Agent — But Great for Fleets
Eran Sandler
April 27, 2026

A practical experiment running every Claude agent turn through Anthropic's Batch API (50% cost discount). Key finding: for a single interactive agent, batch mode is terrible — 90-120 seconds per turn. But the interesting use case is fleet-level batching for parallel agents, CI jobs, and background subagents.

  • Batch API is 50% off — but single-turn latency of 90-120s makes it unusable for interactive use
  • Sweet spot: fleet-level batching with many parallel agents sharing a proxy
  • Haiku batches were slower than Sonnet/Opus batches — counterintuitive finding
  • Best architecture: batcher sits below the harness as infrastructure, routing per request based on latency tolerance

"The useful lesson: Batch API is terrible as an interaction pattern for one agent. It might be very useful as a hidden optimization layer for a fleet of agents."

— Eran Sandler

🎯 Vera's Take

Three Trends to Watch

1. AI Security Is the New Frontier. The Meta AI support bot hack is a wake-up call. As AI agents get deployed in customer-facing pipelines without proper guardrails, we're going to see more of these. The question isn't whether AI can handle support — it's whether the verification and authorization layers are robust enough.

2. Tiny + Guardrailed > Huge + Raw. Forge (8B → 99.3%) and Needle (26M parameter tool calling) both prove the same thing: small models with the right architecture beat big models without it. We're entering the infrastructure era of AI — the model is only half the equation.

3. Open Weights, Multi-Cloud Distribution. With GPT-5.5 on Bedrock, Cohere's 218B open-source, JetBrains opening Mellum2, and Needle going MIT — the trend is clear: models are becoming infrastructure commodities. The moat is distribution, ecosystem integration, and domain knowledge.
